# ДЗ №1: Budget manager

В рамках данного домашнего задания Вам предстоит реализовать небольшое консольное приложение для управления личными финансами.

Наш потенциальный пользователь хочет вносить данные о своём доходе и расходах и получать по ним статистику:
- месячная прибыль/убыток;
- распределение месячных расходов по статьям;
- суммарный размер капитала на конец месяца.

Необходимо реализовать следующие пользовательские сценарии:

## Статьи расхода
### Добавление статьи расхода

```
category add фастфуд
```

Вывод:
```
Добавлено
```

Если такая категория уже есть, всё равно печатаем "Добавлено".

### Переименование статьи расхода
```
# rename dic_costs <old_name> <new_name>
category rename фастфуд еда
```

Если такой категории не было, применяем изменения и печатаем (двойные кавычки должны быть):
```
Категория "фастфуд" переименована на "еда"
```

Если пользователь пытается переименовать категорию, а такая категория уже существует, печатаем:
```
Категория "еда" уже существует. Используйте команду rm
```

### Удаление статьи расхода

```
# Все расходы с этой категорией удалятся
category rm фастфуд
```

```
# Удалить категорию. Всем расходам заменить категорию на другую (даже если её не существует).
category rm фастфуд еда
```

## Добавление дохода

Доход не может быть отрицательным.
```
# Прилетел кэшбэк, целых 49,5 рублей!
income 49,5 28-02-2026

income 49 28-02-2026

income 49,0 28-02-2026
```

## Добавление расхода

Расход не может быть отрицательным.
```
cost фастфуд 555,5 28-02-2026
```

## Просмотр статистики

Вводим:
```
stats
```

Получаем ответ:
```
Ваша статистика по состоянию на данный момент:
Суммарный капитал: 35000.00 рублей
В этом месяце убыток составил 5000.00 рублей
Доходы: 80000.00 рублей
Расходы: 85000.00 рублей

Детализация (категория: сумма):
1. аренда: 45000
2. подарки: 20000
3. подписки: 10000
4. фастфуд: 10000
```

Вводим
```
# Вывод месячной статистики до указанной даты
stats 15-02-2026
```

Получаем ответ
```
Ваша статистика по состоянию на 15-02-2026:
Суммарный капитал: 50000.00 рублей
В этом месяце прибыль составила 10000.00 рублей
Доходы: 40000.00 рублей
Расходы: 30000.00 рублей

Детализация (категория: сумма):
1. подарки: 10000
2. подписки: 10000
3. фастфуд: 10000
```

Обратите внимание:
* Статьи расходов сортируются по убыванию суммы. Если сумма одинаковая, по алфавиту.

* Если расходов в периоде не было - просто не печатаем нумерованный спискок, оставляем строчку с двоеточием.

* Следите за "прибыль составила"/"убыток составил" и "Ваша статистика по состоянию на <>"

## Работа с ошибками
В случае неверных данных, в ответ выводим:
* Если введена неизвестная команда или число аргументов нестандартное: "Неизвестная команда!"
* Если ввели доход или расход меньше или равный нулю: "Значение должно быть больше нуля!"
* Если ввели неверную дату (см. принятые ограничения п.4): "Неправильная дата"
* Если пользователь пытается добавить расход к несуществующей статье расходов, ничего не добавляем и выводим: "Такой категории не знаю"


## Принятые ограничения
1) **Сохранение/чтение данных в файл/бд делать не нужно**. Сериализация/десериализация ждёт нас позже. Исходим из факта, что пользователь при каждом запуске начинает с нуля. То есть нулевой капитал, чистая история и отсутствие источников расходов.
2) **Названия статей расходов (категорий) состоят из одного слова без пробелов**. Пустая строка и/или пробелы не принимаются.
3) **Помните про даты доходов и расходов**. В тестах данные могут быть по разным месяцам, в статистиках ориентируемся только на текущий месяц. Суммарный капитал считаем по всем данным, до указанного дня.
4) **Для дат проверка нестрогая**: день ≤ 31, месяц ≤12, год ≥ 2000. Пока что не работаем с исключениями, так что проверяем всё вручную. А если кто-то и хочет сделать запись 31 февраля, то пусть делает. А вот за 32+ февраля бан.
5) **На вход принимаем и целые, и вещественные числа**.
6) **Удалять функцию `main` нельзя, менять её аргументы тоже.** Эта функция вызывается в тестах.
